📊 How the System Works

┌─────────────┐      ┌──────────────┐      ┌─────────────┐
│   RabbitMQ  │──────>│   Consumer   │──────>│  MongoDB    │
│   Exchange   │      │  (Python)    │      │  Database    │
└─────────────┘      └──────────────┘      └─────────────┘
       ↑                      ↓                        ↓
    You publish           Processes              Stores
    message here         message                 data
                        ↓                        ↓
                   ┌──────────┐            ┌─────────┐
                   │   API    │<──────────│  Browser │
                   │ (FastAPI)│            │  (You)   │
                   └──────────┘            └─────────┘



What Each Component Does:
  1.RabbitMQ Exchange (pods.events): Receives messages and routes them to queues based on routing keys
  2.Queue (gamification.in): Holds messages until consumer processes them
  3.Consumer: Listens to queue, processes messages, awards points, checks for badges
  4.MongoDB: Stores points transactions and member points
  5.API: Lets you query points, transactions, leaderboard

PART 7: Troubleshooting
Problem: "Validation error: franchiseId Field required"
Cause: You forgot to put franchiseId at the top level of the JSON.

Solution: Make sure your JSON has franchiseId at the top level, not just inside payload:

❌ Wrong:

{
  "eventId": "...",
  "payload": {
    "franchiseId": "..."  // Only in payload
  }
}
✅ Correct:

{
  "eventId": "...",
  "franchiseId": "...",  // At top level
  "payload": {
    "franchiseId": "..."  // Also in payload
  }
}



Problem: "No handler found for routing key"
Cause: You used a routing key that doesn't exist.

Solution: Use one of these 4 routing keys:

 . events.attendance.verified
 . training.course.completed
 . training.quiz.passed
 . training.module.completed

Problem: Queue has messages but not being processed
Cause: Consumer is not running or not connected to RabbitMQ.

Solution:

Check if container is running: docker ps
Check logs for errors: docker logs gamification-service
Restart if needed: docker-compose restart gamification-service

Problem: Points not increasing in API
Cause: Message was processed but points weren't updated.

Solution:

Check MongoDB directly (Step 6 and 7)
Check logs for errors
Verify the pointsEarned value is > 0 in the payload
